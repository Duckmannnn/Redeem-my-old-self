# Redeem My Old Self — Recode CP with LeetCode

This repository documents my journey of getting back into problem solving and rebuilding my coding rhythm through LeetCode, mainly using Python.

I have a competitive programming background, mostly with C++.  
Because of that, this repository is not about learning algorithms from zero. Instead, it is about transferring my old CP mindset into clean Python code and gradually increasing the difficulty again.

## Why this repo exists

I used to be more comfortable with competitive programming and algorithmic thinking.  
After some time away from consistent practice, I want to rebuild that version of myself.

This repo is my way of saying:

> I still have the CP foundation.  
> Now I am recoding it in Python, one problem at a time.

## Current Focus

- Getting used to Python syntax for algorithmic problems
- Rebuilding speed and consistency
- Moving from easy problems to stronger medium problems
- Saving problems that feel interesting, elegant, or personally meaningful

## Repository Structure

```txt
easy/       Simple problems for warming up and getting used to Python
medium/     Main practice zone for rebuilding problem-solving rhythm
hard/       Difficult problems that require stronger observation or technique
favorite/   Problems I personally like or find memorable
```

## Background

I have experience with competitive programming, mainly in C++.

Topics I am already familiar with include:

- sorting and searching
- two pointers
- sliding window
- prefix sums
- hash maps
- recursion and backtracking
- graph traversal
- dynamic programming
- greedy algorithms

The goal here is not to make the most beautiful LeetCode repository.  
The goal is to honestly record my process of getting sharper again.

## Notes

I do not write detailed explanations for every problem.

For normal problems, I only save the Python solution.  
For interesting problems, I may add short notes inside the code file or place them in the `favorite/` folder.
